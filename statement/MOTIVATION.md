# MOTIVATION.md

## 1. Название проекта

Прогнозирование HOMO–LUMO gap малых молекул классическими методами машинного обучения

## 2. 10-строчный синопсис

1. HOMO–LUMO gap (разрыв между высшей занятой и низшей незанятой молекулярными орбиталями) — фундаментальное электронное свойство молекул, определяющее оптические, электронные и реакционные характеристики.
2. Прямые квантово-химические расчёты HOMO и LUMO энергий методами DFT или ab initio требуют значительных вычислительных ресурсов даже для малых молекул, что затрудняет высокопроизводительный скрининг.
3. ML-суррогатные модели могут обеспечить быстрые и достаточно точные предсказания HOMO–LUMO gap, выступая в качестве эффективного фильтра перед точными расчётами.
4. В этом проекте фокус на классических (не нейросетевых) ML-моделях, поскольку они менее требовательны к данным, легко интерпретируемы и дают сильный baseline для сравнения.
5. Планируется сравнительное тестирование моделей: Decision Tree, Random Forest, Linear/Ridge/Lasso/ElasticNet регрессии, Gradient Boosting, XGBoost, LightGBM, CatBoost.
6. В качестве признаков рассматриваются стандартные табличные молекулярные дескрипторы: RDKit descriptors, молекулярные фингерпринты (Morgan), простые структурные признаки (число атомов, массу, кольца, водородные связи, полярная поверхность).
7. Целевые значения HOMO–LUMO gap могут быть взяты из открытых датасетов (QM9/QM7b) или внутренних наборов, рассчитанных на уровне DFT.
8. Главная исследовательская цель — оценить, насколько хорошо классические ML-модели способны предсказывать HOMO–LUMO gap малых молекул и какие факторы (выбор модели, набор дескрипторов) влияют на точность.
9. Ожидаемый результат — воспроизводимый pipeline предсказания HOMO–LUMO gap, сравнительная таблица производительности моделей, анализ важности признаков и рекомендации по выбору подхода.
10. Ограничения: качество предсказания ограничено точностью целевых значений (DFT vs эксперимент), риски переобучения на малых выборках, сложность экстраполяции за пределы химического пространства обучающего набора.

## 3. Научная проблема

- HOMO–LUMO gap является фундаментальным молекулярным электронным свойством, напрямую определяющим оптическую поглощение, проводимость, реакционную способность и стабильность молекулы в различных средах.
- Быстрая и точная оценка HOMO–LUMO gap критически важна для скрининга малых молекул в дизайне органических полупроводников, фотокатализаторов, сенсибилизаторов для солнечных элементов и люминофоров.
- Несмотря на появление множества сложных ML-подходов (графовые нейронные сети, ансамбли глубоких моделей), систематическое сравнение простых, интерпретируемых классических моделей для задачи предсказания HOMO–LUMO gap часто отсутствует.
- Интерпретируемость и анализ важности признаков (feature importance) позволяют выявить ключевые структурные фрагменты, влияющие на величину gap, что даёт химическое понимание сверх чисто численного предсказания.

## 4. Практическая проблема

- Полноценные DFT/ab initio расчёты HOMO и LUMO энергий для тысяч молекул требуют дней или недель машинного времени даже на кластерах, что делает их непрактичными для быстрого скрининга.
- Классические ML-модели, обученные на заранее рассчитанных DFT-значениях, могут давать предсказания HOMO–LUMO gap за миллисекунды, обеспечивая быстрый baseline-отбор наиболее перспективных кандидатов.
- Такой baseline особенно полезен на ранних стадиях исследования, позволяя отфильтровать заведомо неподходящие молекулы до запуска дорогих вычислений или более сложных deep learning моделей.

## 5. Основной gap

- В литературе по ML для молекулярных свойств преобладают работы, использующие нейронные сети, графовые нейронные сети или сложные молекулярные представления (например, на основе орбиталей).
- Однако для многих прикладных задач (скрининг, быстрая оценка) необходим понятный и быстрый baseline на классических моделях, который часто отсутствует или не систематизирован.
- Недостаточно ясно, насколько хорошо Random Forest, градиентный бустинг или регуляризованные регрессии работают именно для HOMO–LUMO gap на стандартных наборах молекулярных дескрипторов.
- Не всегда проводится анализ того, какие конкретные дескрипторы вносят наибольший вклад в предсказание HOMO–LUMO gap, что ограничивает химическую интерпретируемость.
- Границы применимости классических моделей для HOMO–LUMO gap часто не исследуются: как модели ведут себя при экстраполяции, при сдвиге химического домена, на малых выборках или при использовании целевых значений, полученных на разных уровнях теории DFT.

## 6. Предлагаемая идея

- Собрать воспроизводимый end-to-end pipeline для предсказания HOMO–LUMO gap, включающий генерацию стандартных молекулярных дескрипторов, обучение нескольких классических ML-моделей, оценку и интерпретацию.
- Использовать только классические (не нейросетевые) модели машинного обучения: Decision Tree, Random Forest, Linear/Ridge/Lasso/ElasticNet регрессии, Gradient Boosting (XGBoost, LightGBM, CatBoost).
- Сравнить производительность моделей на стандартных табличных молекулярных дескрипторах (RDKit, фингерпринты Morgan) для выявления наиболее информативного представления.
- Оценить качество предсказания стандартными метриками регрессии: RMSE, MAE, R², с акцентом на воспроизводимость через кросс-валидацию и отдельный тестовый сплит.
- Для tree-based моделей (Random Forest, XGBoost и др.) провести анализ важности признаков (feature importance) и, при возможности, SHAP.анализ для интерпретации индивидуальных предсказаний.
- Явно разделить задачу предсказания DFT-производного HOMO–LUMO gap от предсказания экспериментального gap, чётко указывая, какой target используется.

## 7. Ожидаемый вклад

### Methodological contribution
- Честное, систематическое сравнение производительности классических ML-моделей для задачи регрессии HOMO–LUMO gap с единым preprocessing, сплитами и метриками.
- Открытый, воспроизводимый pipeline, который можно использовать как baseline для будущих исследований.

### Practical contribution
- Готовый быстрый инструмент для предсказания HOMO–LUMO gap, который можно применять для скрининга малых молекул.
- Практические рекомендации о том, какие модели и дескрипторы использовать в первую очередь для достижения баланса между точностью и скоростью.

### Analytical contribution
- Анализ ошибок моделей: в каких диапазонах HOMO–LUMO gap предсказание работает хорошо, а где систематически ошибается.
- Анализ важных признаков, выявление дескрипторов, наиболее сильно коррелирующих с HOMO–LUMO gap.
- Чёткое описание ограничений применимости построенных моделей (экстраполяция, доменный сдвиг, зависимость от уровня теории DFT).

## 8. Границы проекта

### Входит:
- Малые органические молекулы (размер, аналогичный QM9/QM7b, до ~9 тяжёлых атомов).
- Табличные молекулярные дескрипторы и фингерпринты в качестве признаков.
- Классические supervised regression модели (перечисленные выше).
- Target: electron affinity (вертикальная или адиабатическая), либо её DFT-производный proxy (например, -ε_LUMO), с явным указанием источника.
- Оценка качества предсказания (метрики) и интерпретируемости (важность признаков).

### Не входит:
- Полноценный ab initio расчёт новых значений EA (если автор отдельно не разрешит).
- Графовые нейронные сети и другие deep learning архитектуры как основной метод сравнения.
- Генерация новых молекул (de novo molecular design).
- Утверждения о точном предсказании экспериментальной EA, если target получен из DFT (будет чётко указано, что предсказывается DFT-производная величина).
- Перенос выводов на большие молекулы (полимеры, белки) или другие классы соединений без дополнительной проверки.

## 9. Риски

1. **Target ambiguity**: EA может означать экспериментальное сродство к электрону, вертикальное EA, адиабатическое EA или proxy через орбитальные энергии (например, -ε_LUMO). Разные определения дают разные численные значения. *Мит:* явно зафиксировать и документировать определение target, используемое в проекте.

2. **Dataset leakage**: Молекулы в наборе данных могут быть структурно схожи (изомеры, гомологи). Если не аккуратно разделить train/test, может возникнуть leakage, завышающий оценку обобщающей способности. *Мит:* использовать стратифицированные сплиты на основе молекулярных фингерпринтов или scaffold splitting.

3. **Descriptor dependence**: Качество модели может сильнее зависеть от выбора набора дескрипторов, чем от выбора алгоритма ML. Плохой набор признаков не даст хорошего предсказания даже с самой сложной моделью. *Мит:* тестировать несколько различных наборов дескрипторов и сравнивать их влияние.

4. **Overclaiming**: Высокий R² на DFT-производном target не означает точного предсказания экспериментальной EA. Модель может просто хорошо воспроизводить систематическую ошибку DFT. *Мит:* чётко указывать в выводах, что предсказывается именно DFT-значение, и обсуждать возможные расхождения с экспериментом.

5. **Domain shift**: Модель, обученная на одном химическом пространстве (например, насыщенные углеводороды), может катастрофически плохо работать на молекулах другого класса (например, ароматические гетероциклы). *Мит:* анализировать химическое разнообразие обучающей выборки и тестировать на внешних наборах, если они доступны.

6. **Small-data risk**: Если доступный датасет с EA мал (сотни молекул), сложные модели (например, градиентный бустинг с большим числом деревьев) могут легко переобучаться. *Мит:* использовать строгую кросс-валидацию, регуляризацию и отдавать предпочтение более простым моделям в условиях малых данных.

## 10. Current project state

The project has advanced beyond the initial proposal stage and already possesses primary experimental results.

- The dataset for electron affinity prediction is already available in `dopanti_paper/compact_data_FHI_575molecules.csv`. It contains 575 molecules (576 rows including header) with SMILES, InChi keys, and target columns EA (eV) and BG (eV). The target EA values are **DFT‑calculated** (not experimental), and the dissimilarity criterion for the top‑20 analysis is **finalized** as the maximum Tanimoto similarity based on Morgan fingerprints.

- Initial classical ML experiments have already been performed in `dopanti_paper/dopanti_small_mols_final_exps.ipynb`.

- The tested models include Linear Regression, Decision Tree, Random Forest, Support Vector Regression (SVR) and XGBoost (Gradient Boosting). The exact names in the code correspond to `LinearRegression`, `DecisionTreeRegressor`, `RandomForestRegressor`, `SVR` (from sklearn.svm) and `XGBRegressor`.

- The evaluation includes repeated cross-validation with MSE and R² metrics. The results are aggregated across repeats and folds, with mean ± standard deviation reported for each model.

- In addition to standard validation on the full validation set, the project includes a stress‑test‑like evaluation on the top‑20 chemically dissimilar molecules from each validation split. This makes the project focused not only on average predictive quality, but also on generalization to less similar molecules.

- The concrete result tables are stored in CSV files: `models_cv_raw.csv` (per‑fold metrics), `models_cv_repeat_summary.csv` (aggregated per repeat), `models_cv_summary_full.csv` and `models_cv_summary_compact.csv` (overall mean ± std for each metric), and `models_cv_ranking.csv` (ranking by R²).

- The preliminary results show that XGBoost achieves the highest R² on the full validation set (0.793 ± 0.077) and on the top‑20 dissimilar molecules (0.684 ± 0.286). Linear Regression, SVR and Random Forest follow closely. Decision Tree exhibits the lowest average R² and the largest drop on the dissimilar subset.

Thus the project already has a reproducible baseline pipeline, concrete performance numbers, and a first assessment of generalization ability. The next steps are to analyse feature importance, possible data leakage, and the dependence on the chosen molecular descriptors.

## 11. Список литературы для следующего шага

- **R1 (Pereira 2023)** — центральная статья по ML-предсказанию IP и EA по DFT-данным; содержит результаты для Random Forest, SVM, MLP, LightGBM. Нужна для сравнения с нашими классическими моделями и для выбора метрик/препроцессинга. *Статус: PDF недоступен (ConnectTimeout).*
- **R2 (Ramakrishnan et al. 2014)** — описание датасета QM9, основного источника DFT-свойств для малых молекул. Нужна для описания источника данных и понимания структуры датасета. *Статус: PDF недоступен (HTTP 522).*
- **R3 (Montavon et al. 2013)** — ранняя работа по ML для электронных свойств, включая EA; полезна для исторического контекста и обзора эволюции подхода. *Статус: скачан (17 стр.).*
- **R4 (Wu et al. 2017/18)** — бенчмарк MoleculeNet; содержит описание датасетов (QM9, QM7b) и baseline-методик оценки. Нужна для стандартных сплитов, метрик и воспроизводимости. *Статус: скачан (39 стр.).*
- **R5 (Valdés & Tchagang 2024)** — анализ структуры и сложности датасетов QM7b и QM9; полезен для понимания ограничений данных и кластерной структуры. *Статус: PDF недоступен (HTTP 403).*
- **R6 (Ullah et al. 2024)** — свежий обзор квантово-химических датасетов; даёт контекст для выбора данных и обсуждения ограничений разных уровней теории. *Статус: скачан (62 стр.).*
- **R7 (Karandashev et al. 2022)** — статья о продвинутых молекулярных представлениях (орбитальных) для ML; контрастный пример сложных подходов, полезный для обсуждения выбора дескрипторов. *Статус: PDF недоступен (HTTP 403).*
- **R8 (Fediai et al. 2023)** — работа с уточнёнными GW-значениями орбитальных энергий для QM9; важна для обсуждения точности DFT-целей и ограничений базового уровня теории. *Статус: скачан (20 стр.).*
- **R9 (Zhan et al. 2025)** — свежая прикладная работа по EA/IE с использованием классических моделей (CatBoost, XGBoost и др.); современный контекст и пример инженерного применения. *Статус: PDF недоступен (HTTP 403/paywall).*
