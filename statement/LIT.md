# Литературный обзор

## 1. Цель литературного обзора

Цель — систематизировать существующие подходы к прогнозированию энергии сродства к электрону (EA) малых молекул, с акцентом на методы машинного обучения. Обзор должен выявить gap в исследованиях: недостаточное внимание к классическим ML-моделям для EA, несистематическое сравнение дескрипторов, ограниченная интерпретируемость. На основе доступных локальных источников (PDF в `materials/references/`) и информации из `MOTIVATION.md` структурируем текущее состояние и определим место проекта.

## 3. Классификация подходов

### Группа A: Высокоточные квантово-химические методы (референсные)
- **Методы**: CCSD(T), GW, CASPT2, CBS-экстраполяция
- **Точность**: высокая (~0.01–0.05 эВ)
- **Скорость**: очень медленная (часы–дни на молекулу)
- **Применение**: создание референсных данных, калибровка DFT/ML
- **Примеры работ**: R8 (GW для QM9)
- **Роль в нашем проекте**: источник высокоточных target-значений для оценки ограничений DFT-данных

### Группа B: Методы функционала плотности (DFT) — основной источник данных
- **Методы**: B3LYP, PBE0, ωB97X-D, M06-2X
- **Точность**: средняя (~0.1–0.3 эВ), зависит от функционала
- **Скорость**: умеренная (минуты–часы)
- **Применение**: генерация больших датасетов для ML
- **Примеры работ**: R2 (QM9 на DFT), R1 (предсказание DFT-значений ML)
- **Роль в нашем проекте**: основной источник target-данных (если используется QM9); baseline точности

### Группа C: Классические ML-модели на табличных дескрипторах
- **Методы**: Random Forest, Gradient Boosting (XGBoost, LightGBM, CatBoost), линейные регрессии, kernel ridge regression
- **Точность**: потенциально близка к DFT (~0.1–0.2 эВ)
- **Скорость**: быстрая после обучения (миллисекунды на предсказание)
- **Применение**: высокопроизводительный скрининг, интерпретация признаков
- **Примеры работ**: R1 (RF, LightGBM), R3 (kernel ridge), R9 (CatBoost, XGBoost)
- **Роль в нашем проекте**: основной фокус проекта; сравнение производительности, анализ важности признаков

### Группа D: Глубокое обучение и графовые нейронные сети (GNN)
- **Методы**: GNN, transformer, нейронные сети на молекулярных графах или орбитальных представлениях
- **Точность**: потенциально высокая, но требует больших данных
- **Скорость**: умеренная (обучение долгое, инференс быстрый)
- **Применение**: скрининг с автоматическим извлечением признаков
- **Примеры работ**: R7 (орбитальные представления), R4 (MoleculeNet включает GNN)
- **Роль в нашем проекте**: контрастная группа; не входит в границы проекта, но упоминается в gap analysis

### Группа E: Полуэмпирические и эмпирические методы
- **Методы**: групповые вклады, линейные корреляции, QSPR
- **Точность**: низкая–средняя (~0.2–0.5 эВ)
- **Скорость**: очень быстрая (миллисекунды)
- **Применение**: прикидочные оценки, образование

## 4. Gap analysis

### Что уже хорошо решено
- Существуют большие датасеты DFT-свойств малых молекул (QM9, QM7b) — R2, R6
- Бенчмарки ML-методов на молекулярных свойствах (MoleculeNet) — R4
- ML-предсказание EA на DFT-данных показало потенциал (R1, R3, R9)
- Уточнённые GW-данные доступны для оценки погрешностей DFT (R8)

### Что решено частично
- Сравнение классических ML-моделей для EA: есть отдельные работы (R1, R9), но нет систематического сравнения на едином пайплайне
- Интерпретация важности признаков для EA: упоминается в R1 (LightGBM feature importance), но не глубоко анализируется
- Влияние выбора дескрипторов на точность EA prediction: обсуждается в общем виде (R6, R7), но не для EA специфически

### Что остаётся открытым (gap)
1. **Систематическое сравнение классических ML-моделей** для EA: нет работы, где бы на одном датасете, с одинаковыми сплитами и метриками сравнивались Decision Tree, Random Forest, линейные регрессии, XGBoost, LightGBM, CatBoost.
2. **Анализ зависимости точности от типа дескрипторов** для EA: RDKit vs. Mordred vs. fingerprints vs. простые структурные признаки.
3. **Интерпретируемость моделей EA**: SHAP-анализ или feature importance, связывающий дескрипторы с химической интуицией.
4. **Оценка границ применимости** классических моделей для EA: экстраполяция, доменный сдвиг, малые выборки.
5. **Чёткое разделение** предсказания DFT-производного EA vs. экспериментального EA в литературе по ML.

### Где находится место нашего проекта
- В **заполнении gap 1 и 2**: систематическое сравнение классических моделей и дескрипторов для EA.
- В **частичном заполнении gap 3**: анализ важности признаков для tree-based моделей.
- В **явном указании gap 5**: чёткое документирование, что target — DFT-производная величина (если используется QM9).
- В **создании воспроизводимого пайплайна**, который можно использовать как baseline для будущих исследований.

## 7. Relevance to current project

Литература, собранная в обзоре, непосредственно связана с конкретной задачей проекта:

- **R1 (Pereira 2023)** является наиболее близкой работой, поскольку рассматривает prediction of ionization potential and electron affinity using DFT-derived targets and machine learning (Random Forest, SVM, MLP, LightGBM). Наш проект расширяет это сравнение, добавляя более широкий набор классических моделей (Linear Regression, Decision Tree, Random Forest, Gradient Boosting, SVR) и вводя анализ качества на химически непохожих молекулах. **Наш target также DFT‑calculated, что обеспечивает полную сопоставимость с методологией R1.**

- **QM9/QM7b-related источники (R2, R3, R5, R6, R8)** служат контекстом для small‑molecule quantum‑chemical property prediction. Хотя наш датасет не является QM9 (используется внутренний набор из 575 молекул), обсуждение ограничений DFT, точности target‑значений и кластерной структуры данных остаётся релевантным.

- **MoleculeNet (R4)** предоставляет общий benchmark‑контекст для molecular machine learning, устанавливая стандартные сплиты и метрики. В нашем проекте используется собственная стратегия валидации (Repeated K‑Fold 10×3), но сравнение с бенчмарками MoleculeNet может быть частью обсуждения.

- **Работы по molecular descriptors / representations (R6, R7)** обосновывают выбор табличных дескрипторов и фингерпринтов. В ноутбуке `dopanti_small_mols_final_exps.ipynb` используются:
  - Линейные дескрипторы RDKit (208 признаков)
  - Структурные дескрипторы (asphericity, planarity)
  - Энергия запрещённой зоны (BG) как дополнительный признак
  - Фингерпринты Morgan (радиус 2, 2048 бит) для расчёта сходства
  Это соответствует рекомендациям литературы о комбинировании различных типов признаков для электронных свойств.

- **Отличие нашего проекта** заключается в трёх аспектах:
  1. **Фокус на классические supervised regression модели** — Linear Regression, Decision Tree, Random Forest, Gradient Boosting, SVR — без привлечения глубокого обучения или графовых нейронных сетей.
  2. **Анализ качества не только на обычной многоповторной кросс‑валидации, но и на top‑20 chemically dissimilar validation molecules in each split**. Это позволяет оценить, насколько модели способны экстраполировать на химически менее похожие структуры.
  3. **Использование внутреннего датасета малых молекул (575 samples)**, что даёт возможность проверить поведение моделей на реалистичном, но умеренном по размеру наборе данных.

## 8. Gap analysis (уточнение)

На основе детального анализа проведённых экспериментов gap можно уточнить следующим образом:

- **В литературе** есть множество работ по prediction of molecular electronic properties, включая electron affinity (R1, R3, R9). Однако часто акцент делается на более сложных molecular representations, neural networks, graph‑based models или общих benchmark‑сравнениях (R4, R7). Систематическое сравнение простых классических моделей для EA остаётся фрагментарным.

- **Для практического screening** полезно отдельно понять, насколько хорошо работают простые классические модели: Linear Regression, Decision Tree, Random Forest, Gradient Boosting. Эти модели быстрее в обучении и инференсе, более интерпретируемы и требуют меньше вычислительных ресурсов, что важно для высокопроизводительного скрининга.

- **Важно оценивать** не только средние метрики на random splits / repeated cross‑validation, но и поведение модели на молекулах, которые химически менее похожи на training set. Если модель хорошо работает только на молекулах, структурно близких к обучающим, её практическая полезность для скрининга новых химических пространств ограничена.

- **Поэтому отдельный анализ top‑20 dissimilar molecules from validation split** является ключевым элементом проекта. В ноутбуке реализован следующий критерий dissimilarity:
  - Для каждой молекулы в validation fold вычисляется **максимальное сходство Tanimoto** с любой молекулой из training fold на основе фингерпринтов Morgan (радиус 2, 2048 бит).
  - Молекулы сортируются по возрастанию этого сходства (т.е. от наименее похожих к более похожим).
  - Выбираются **20 молекул с наименьшим сходством** (top‑20 dissimilar). **(Критерий dissimilarity окончательный, подтверждён автором.)**
  - Метрики (RMSE, R²) вычисляются отдельно для этой подвыборки.

- **Обнаруженный gap**: отсутствие работ, где бы систематически оценивалось качество классических ML‑моделей для EA **одновременно** на полном validation set и на специально выделенных химически непохожих молекулах. Наш проект заполняет этот gap, предоставляя сравнение пяти классических моделей с двойной оценкой качества.
